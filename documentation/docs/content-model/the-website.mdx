# The Website

The Website section covers public-facing research pages and the TEHS Quarterly digital archive.

## Curated Essays & Overviews

Use a **Curated Essay** for long-form modern research, overviews, or interactive pages—not for raw transcriptions of historical clippings.

Key fields:

- **Page title** and required **URL slug** (generated from the title)
- **Townships** — geographic scope of the essay
- **Page content & layout canvas** — Portable Text with:
  - Text blocks and **Uploaded Images** (caption, alt, and `displayMode`: full / left / right)
  - **Interactive map modules** (`mapYear`, embedded `mapUrl`)
  - **Nested navigation portals** linking to other curated essays

Related: [Publish a curated essay](../common-tasks/publish-curated-essay.mdx)

## TEHS Quarterly Articles

A **TEHS Quarterly Article** stores the full digitized text of a Quarterly piece, plus publication metadata and entity links.

Key fields:

- **Publication Details** — title, author name, volume, issue, publication date (text), and start page for table-of-contents ordering
- **Article Content** — Portable Text body with:
  - Standard text blocks (paragraphs, headings)
  - Inline images (caption and alt text)
  - **Original Print Page Break** blocks to preserve academic pagination (e.g., "Page 3") without disrupting reading flow
- **Tagged Entities** — references to properties and people mentioned in the article

Archive documents (primary sources and historical images) can still reference these under **Research References**.

### Dynamic table of contents

Because articles are structured data rather than static HTML, the website can generate volume indexes on the fly. For example, to recreate the Volume 15 table of contents:

```groq
*[_type == "quarterlyArticle" && volume == 15] | order(startPage asc) {
  title,
  authorText,
  startPage,
  publishedDate
}
```
